# AGENTS.md

## Что это

Внутренний shared slice библиотеки.

## Назначение

`shared` хранит низкоуровневые общие части, которые могут использоваться разными срезами, но сами не являются public UI API.

Сейчас фактически используется `src/shared/styles/shadow` для общего SCSS helper `shadow-size.scss`.

## Как реализован

- У `src/shared` нет public `index.ts`.
- Shared-код подключается только внутренними относительными imports.
- Текущие дочерние части:
  - `styles/shadow` — общий SCSS helper для shadow sizing.

## Public API и локальные файлы

- Public API отсутствует.
- `shared` не должен экспортироваться из `@sellgar/kit`, `@sellgar/kit/icons` или feature entrypoints.
- Если здесь появятся утилиты, они должны оставаться внутренними до отдельного решения о public API.

## Фактическое поведение

- `shared` не владеет React-компонентами.
- `shared` не должен знать о product-level компонентах, features или icon API.
- Общая часть переносится сюда только если она реально используется несколькими соседними или разными срезами.

## Зависимости

- Разрешены низкоуровневые зависимости без knowledge о `core`, `icons` и `features`.
- Запрещены imports из `src/core`, `src/icons`, `src/features`.

## Правила изменений

- Не переносить сюда helper только ради “чистоты” каталога. Сначала должно быть реальное повторное использование.
- Не создавать public barrel `src/shared/index.ts` без отдельного архитектурного решения.
- Не добавлять React-компоненты в `shared`.
- Если общий код относится к одному компоненту, он остаётся рядом с этим компонентом.

## Риски и точки внимания

- `shared` легко превратить в свалку. Каждый новый файл должен иметь понятного потребителя и уровень абстракции.
- Зависимость `shared -> core/icons/features` нарушает слои и может создать cycles.

## Технический долг

- Папка `src/shared/utils` зарезервирована, но сейчас не используется. Если утилит не появится, каталог стоит удалить отдельной cleanup-правкой.

## Проверка

- После изменения SCSS helpers проверять `../../node_modules/.bin/tsc -p tsconfig.json --noEmit --incremental false`, если менялись imports.
- Для изменений, влияющих на CSS output, запускать `yarn build` в `library/kit`.
